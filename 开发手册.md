## 分支规则

| 分支 | 用途 | 规则 |
|------|------|------|
| `main` | 生产环境，同事在用 | 只改稳定代码，不推实验功能 |
| `trace` | 溯源+向量检索 | 可随意改，不影响 main |

**切换分支前先 `git stash`**，否则本地修改会被带走。

## 日常操作

### 启动生产服务器
```bash
python app.py    # 端口 5000，同事用
```

### 启动测试服务器
```bash
python app.py    # 改 app.py 端口为 5001 再启动
```

### 重启服务器
```bash
# 任务管理器 → 结束 python.exe → 重新 python app.py
```
`use_reloader=False` 不会自动重载，必须手动重启。

### 数据清空
```python
python -c "
from app import create_app
from models import db, ReviewField, ModuleAssignment, Project
app = create_app()
with app.app_context():
    ReviewField.query.delete()
    ModuleAssignment.query.delete()
    Project.query.delete()
    db.session.commit()
    db.session.execute(db.text('VACUUM'))
    db.session.commit()
"
```

### 备份数据库
```python
python backup.py    # 备份到 backups/ 目录，保留最近 48 个
```

## CSS 踩坑

1. **双类选择器提高优先级**：`.et-btn` 有 `background: #fff`，按钮用 `.et-btn.btn-save` 覆盖
2. **`ms-auto` 内的按钮垂直堆叠**：加 `d-flex gap-1` 解决
3. **卡片高度限制**：`.panel-body` 和 `.formatted-text` 各有 `max-height`，改一个不管用

## JS 踩坑

1. **不要用 `?.` 可选链**：老浏览器不支持
2. **`onclick` 属性用单引号**：`tojson` 输出双引号，和 HTML 双引号冲突
3. **Quill 初始化必须在可见元素上**：用 `position:absolute; left:-9999px` 临时显示
4. **Quill 颜色兼容**：`formatText(index, length, 'color', '#C00000')` 需要先扩展颜色白名单

## Jinja 踩坑

1. **`tojson` 过滤器输出双引号**：和 `onclick="..."` 冲突，改 `onclick='...'` 或预计算索引
2. **`loop.parent`**：内层循环访问外层用 `loop.parent.index0`，不是 `loop.parent.loop.index0`
3. **`namespace` 对象**：`{% set ns = namespace(val=0) %}` 在旧版 Jinja 不支持

## git 踩坑

1. **cherry-pick 前先确认在哪个分支**：`git branch` 查看
2. **push 失败检查分支**：`trace` 分支没有 upstream，需要 `git push --set-upstream origin trace`
3. **离线文件不要提交**：`.gitignore` 已配 `Payroll_Practical_Handbook_*` 和 `uploads/*`
