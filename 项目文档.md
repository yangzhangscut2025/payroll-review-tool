> 分支：`excel-format-v2` | 日期：2026-07-23

## 一、项目概述

薪酬合规审阅工具是一个团队协作的 Web 应用，用于对各国薪酬合规规则进行逐条审阅、标注和导出。

### 核心流程

```
上传 Excel → 分配模块给校验人 → 审阅工作台逐条审阅 → 导出审阅结果
```

### 用户角色

| 角色 | 权限 | 用户 |
|------|------|------|
| 负责人 | 创建项目、上传文件、分配模块、编辑所有内容、删除项目 | 全部用户 |

## 二、三格式 Excel 支持

系统支持三种 Excel 格式，上传时自动识别。

### v1 格式（13 列，3 对照对）

| 对照对 | 内容列 | 参考依据列 |
|--------|--------|-----------|
| 官方 | 实务内容（官方） | 参考依据（官方） |
| 行业通用 | 实务内容（行业通用） | 参考依据（行业权威） |
| 内部口径 | 实务内容（内部口径） | 参考依据（行业常规） |

### v2 格式（7 列，2 对照对）

| 对照对 | 内容列 | 参考依据列 |
|--------|--------|-----------|
| 官方规则 | 官方规则 | 官方网站 |
| 行业通用 | 行业通用 | 权威网站 |

**识别规则**：首列表头为「标题Ⅰ」→ v2 格式。

### v3 格式（10 列，2 对照对）

| 对照对 | 内容列 | 参考依据列 |
|--------|--------|-----------|
| 官方 | 实务内容（官方） | 参考依据（官方） |
| 行业通用 | 实务内容（行业通用） | 参考依据（行业权威） |

**识别规则**：首列表头为「l1_标题」且无「实务内容（内部口径）」列 → v3 格式。

### v3 格式 Excel 结构

| l1_标题 | l1_说明 | l2_标题 | l2_说明 | 实务内容（官方） | 实务内容（行业通用） | 参考依据（官方） | 参考依据（行业权威） | 状态 | (空) |
|-------|-------|------|------|---------|---------|---------|---------|------|------|
| 一级模块 | 说明 | 二级模块 | 审阅范围 | 实务内容 | 实务内容 | 链接 | 链接 | 状态 | - |

## 四、AI Prompt 面板

### 功能说明

审阅工作台右侧的 Prompt 面板，可以自动生成 AI 核对提示词，方便将当前模块的内容和参考链接发送给 AI 助手进行交叉验证。

### 按钮功能

| 按钮 | 功能 |
|------|------|
| 🔄 刷新 | 重新生成 Prompt（切换模块后需手动刷新） |
| 📋 复制 | 一键复制 Prompt 到剪贴板 |
| 🚀 发送 | 直接调用 DeepSeek API（**不推荐**，见下文） |
| × | 关闭面板 |

### ⚠️ 推荐使用方式：复制到豆包

**不要使用「🚀 发送」按钮。** 原因：

1. **效果差**：API 调用缺少对话上下文，输出质量明显不如直接在豆包/DeepSeek 对话框中粘贴 Prompt
2. **耗费 API**：每次调用消耗 token，产生费用
3. **无法追问**：API 只返回一次结果，不能在对话中追问修正

**推荐流程**：

```
1. 审阅工作台 → 切换模块 → 点 🤖 Prompt
2. 点 📋 复制
3. 打开 豆包 或 DeepSeek 对话框
4. 粘贴 → 发送
5. 根据 AI 返回结果，回到审阅工作台标注和修改
```

### Prompt 模板结构

生成的 Prompt 包含三部分：

- **核对规则**（9 条）：判断标准、拆分处理、数据核验、上下文检查、链接矛盾处理、否定记录、推断标注、时效风险、外部引用检查
- **输出格式**：统一的 `判断/溯源/说明` 模板
- **待核对信息 + 参考链接**：当前模块的内容文本和链接列表（v2 格式会自动合并「官方网站」和「权威网站」两份链接，并标注来源）

### 费用估算

基于 DeepSeek 定价：输入 ¥1/百万 token + 输出 ¥2/百万 token。

每次典型调用约 2000 prompt + 500 completion ≈ ¥0.003，成本极低。

## 七、导出审阅结果

点击项目详情页「📥 导出审阅结果」按钮。

- 预览弹窗显示 Sheet 列表、字段数、翻译量（仅统计有审阅标记的改动）、费用、耗时
- 确认后异步导出，实时进度条，完成自动下载
- 中文 sheet：直接填审阅内容
- 英文 sheet：有审阅标记的字段翻译为英文，无标记的直接用英文原文
- 翻译失败自动回退英文原文，黄色背景标记
- 文件名格式：`{项目名}_审阅结果_{日期}.xlsx`

**⚡ 快速导出**：直接下载中文版，不翻译不预览，适合快速检查。

## 九、已修复问题记录（2026-07-21）

### 布局与交互

| 问题 | 原因 | 修复 |
|------|------|------|
| Prompt 面板被挤出三列 | review.html 第245行 `</div>` 过早关闭 row | 删除多余 `</div>`，mainCol 宽度改为 JS 动态计算 |
| 折叠侧栏后宽度不更新 | `toggleSidebar()` 未更新 mainCol max-width | 新增 `updateMainColWidth()` 统一计算 |
| Prompt 面板不感知侧栏 | `togglePrompt()` 硬编码 180px | `getSidebarWidth()` 动态读取实际宽度 |
| 刷新后状态丢失 | `restoreViewState()` 未恢复 Prompt 面板和宽度 | 恢复 Prompt 可见性 + `updateMainColWidth()` + `beforeunload` 保存 |
| 切换模块后面板空白 | `sessionStorage` 存的 tab 索引跨项目不验证 | `restoreViewState()` 校验 `savedTab < tabs.length` |
| 树节点点太快无响应 | `navigateToIdx` 在外部 JS 里，加载慢时未定义 | 内联到模板 `<script>` 块 |

### 数据与导出

| 问题 | 原因 | 修复 |
|------|------|------|
| 参考链接有时不显示 | `initAllRefTexts()` 用 `getAttribute` 判空，空字符串被跳过 | 去掉判空，直接调用 `renderRefText` |
| `link_urls` 从未持久化 | 每次页面加载临时提取 URL，不存库，刷新后丢失 | 上传时/保存时提取 URL 写入 `link_urls` 字段 |
| 重新上传后数据重复 | `parse_excel()` 只新增不删旧 | 上传前 `ReviewField.query.delete()` 清旧数据 |
| 导出线程不安全 | `project` 对象在后台线程中过期 | `app_context` 内重新获取 + `db.session.get()` |
| `_export_progress` 内存泄漏 | 全局字典无过期清理 | 1 小时 TTL 自动清理 |
| 并发导出无保护 | 同项目可发起多个导出 | 检查已有活跃任务，复用 |
| 翻译失败静默 | 无 API key 或翻译异常时无提示 | 返回警告信息，进度条显示 ⚠️ |
| 上传文件覆盖 | 固定文件名 `project_{id}.xlsx` | 加时间戳 `project_{id}_{timestamp}.xlsx` |
| 翻译用量未记录 | 导出翻译 API 调用未写入 `AIUsageLog` | token 用量回传并在 `app_context` 中记录 |
| 项目删除不可恢复 | 物理删除三张表记录 | 改为软删除（设 `deleted_at`） |
| 翻译全量无差异 | Quill 自动包 `<p>` 标签，所有字段被判为"有改动" | 检查审阅标记（`<span style>`/`<s>`/`<strong>`），只有真正标注的才翻译 |
| 翻译失败整批丢弃 | AI 返回条目数不一致时整语言翻译被丢弃 | 逐批重试（最多 3 次），缺的用英文原文回退 |
| 翻译缺英文参考 | 中文审阅内容直接翻译，术语可能不一致 | Prompt 同时发送英文原文作为术语参考 |
| 翻译失败无标记 | 回退中文原文无法区分 | 黄色背景标记（`FFFACD`），一眼可识别 |
| 快速导出误触 | 无确认直接下载 | 加 confirm 弹窗确认 |
| 预览 Token 估算偏低 | 未计入英文原文参考的 token | 预览估算加入英文原文字符数 |
| 无改动时仍显示翻译进度 | 全部字段未改动时进度条仍显示"翻译英文" | 无改动时跳过翻译，进度条显示"无需翻译" |
| 项目列表页导出链接报错 | `export_result` 路由被替换后 `index.html` 引用失效 | 改用直接 URL 链接 |
| 服务器导出 FileNotFound | 数据库 `file_path` 存的是 Windows 路径，Ubuntu 找不到 | 批量更新为 Linux 路径 |
| v1 项目导出空文件 | 导出按 sheet 名 `zh_Chinese` 匹配，v1 项目 sheet 名不同 | 改为按 sheet 位置（第一个=中文，第二个=英文） |
| 无效链接导出无区别 | 链接状态只在审阅页面 JS 渲染，导出时丢失 | 无效链接红色+删除线，附纠正链接和状态备注 |
| 模块树无状态指示 | 圆点全灰色，不知道哪个模块审完了 | 后端聚合 l2 状态，前端圆点绿/黄/灰三色 |
| 圆点被挤压 | flex 布局中 l2 名称过长时圆点变形 | 加 `flex-shrink: 0` 防止挤压 |

### 渲染与兼容性

| 问题 | 原因 | 修复 |
|------|------|------|
| v2 格式说明列读错 | l2/说明列号硬编码，v2 下应为 col2/col3 | 根据 `format_version` 动态选择列号 |
| 背景色覆盖字体色 | `background-color` CSS 属性错误写入 `color` | 删除该代码块 |
| fetchWithRetry 二次读 body | `resp.json()` 消费了 body，返回后调用方再读失败 | `resp.clone().json()` 克隆后读取 |
| HTML 嵌套标签翻译错乱 | 直接让 AI 保留标签，语序变化后错位 | 占位符保护：`<span>` → `{__T0__}`，翻译后还原 |
| `data.db` 被 Git 追踪 | 数据库文件包含敏感数据 | `.gitignore` + `git rm --cached` |
| JS 语法错误 (行1377) | 模块名含特殊字符，Jinja `"{{ value }}"` 拼入 JS 时未转义 | 全局改用 `{{ value|tojson }}`，自动处理引号和特殊字符 |
| 8 人并发 404 | Flask 自带单线程服务器，请求排队超时 | 换 Waitress 生产服务器，16 线程 |
