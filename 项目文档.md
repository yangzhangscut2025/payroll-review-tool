> 分支：`excel-format-v2` | 日期：2026-07-21

## 一、项目概述

薪酬合规审阅工具是一个团队协作的 Web 应用，用于对各国薪酬合规规则进行逐条审阅、标注和导出。

### 核心流程

```
上传 Excel → 分配模块给校验人 → 审阅工作台逐条审阅 → 导出审阅结果
```

### 用户角色

| 角色 | 权限 | 用户 |
|------|------|------|
| 负责人 | 创建项目、上传文件、分配模块、编辑所有内容、删除项目 | Zora, Jack |
| 校验人 | 审阅自己负责的模块、更改状态、编辑内容 | Elara, Hailey, kevin, Leon, Mercury, Touyi, Yee |

## 二、双格式 Excel 支持

系统支持两种 Excel 格式，上传时自动识别。

### v1 格式（13 列，3 对照对）

| 对照对 | 内容列 | 参考依据列 |
|--------|--------|-----------|
| 官方 | 实务内容（官方） | 参考依据（官方） |
| 行业通用 | 实务内容（行业通用） | 参考依据（行业权威） |
| 内部口径 | 实务内容（内部口径） | 参考依据（行业常规） |

### v2 格式（7 列，2 对照对）

| 对照对 | 内容列 | 参考依据列 |
|--------|--------|-----------|
| 官方规则 | 官方规则 | 官方网站 |
| 行业通用 | 行业通用 | 权威网站 |

**识别规则**：首列表头为「标题Ⅰ」→ v2 格式，否则 → v1 格式。

### v2 格式 Excel 结构

| 标题Ⅰ | 标题Ⅱ | 说明 | 官方规则 | 行业通用 | 官方网站 | 权威网站 |
|-------|-------|------|---------|---------|---------|---------|
| 一级模块 | 二级模块 | 审阅范围说明 | 实务内容 | 实务内容 | 链接 | 链接 |

## 四、AI Prompt 面板

### 功能说明

审阅工作台右侧的 Prompt 面板，可以自动生成 AI 核对提示词，方便将当前模块的内容和参考链接发送给 AI 助手进行交叉验证。

### 按钮功能

| 按钮 | 功能 |
|------|------|
| 🔄 刷新 | 重新生成 Prompt（切换模块后需手动刷新） |
| 📋 复制 | 一键复制 Prompt 到剪贴板 |
| 🚀 发送 | 直接调用 DeepSeek API（**不推荐**，见下文） |
| × | 关闭面板 |

### ⚠️ 推荐使用方式：复制到豆包

**不要使用「🚀 发送」按钮。** 原因：

1. **效果差**：API 调用缺少对话上下文，输出质量明显不如直接在豆包/DeepSeek 对话框中粘贴 Prompt
2. **耗费 API**：每次调用消耗 token，产生费用
3. **无法追问**：API 只返回一次结果，不能在对话中追问修正

**推荐流程**：

```
1. 审阅工作台 → 切换模块 → 点 🤖 Prompt
2. 点 📋 复制
3. 打开 豆包 或 DeepSeek 对话框
4. 粘贴 → 发送
5. 根据 AI 返回结果，回到审阅工作台标注和修改
```

### Prompt 模板结构

生成的 Prompt 包含三部分：

- **核对规则**（9 条）：判断标准、拆分处理、数据核验、上下文检查、链接矛盾处理、否定记录、推断标注、时效风险、外部引用检查
- **输出格式**：统一的 `判断/溯源/说明` 模板
- **待核对信息 + 参考链接**：当前模块的内容文本和链接列表（v2 格式会自动合并「官方网站」和「权威网站」两份链接，并标注来源）

## 五、项目信息编辑

项目详情页（`/projects/<id>`）支持编辑项目名称和国家。

- 点击标题栏右侧 **✎ 编辑** 按钮
- 项目名称和国家变为输入框
- 修改 → 保存 → 原地生效
- 仅项目负责人可编辑

## 六、AI 用量统计

导航栏「📊 AI用量」入口，可查看所有人员的 API 调用记录。

### 统计维度

| 维度 | 内容 |
|------|------|
| 总览卡片 | 总调用次数、总 Token 用量、估算费用、人均调用 |
| 按人员统计 | 每次用户的调用次数和 Token 消耗 |
| 按项目统计 | 每个项目的 AI 调用汇总 |
| 最近记录 | 最近 50 条调用详情（时间/用户/项目/模块/模型） |

### 费用估算

基于 DeepSeek 定价：输入 ¥1/百万 token + 输出 ¥2/百万 token。

每次典型调用约 2000 prompt + 500 completion ≈ ¥0.003，成本极低。

## 七、导出审阅结果

点击项目详情页「📥 导出审阅结果」按钮。

- 在原 Excel 每列右侧插入「校验列」
- 保留 Quill 编辑器的富文本格式（颜色、加粗、删除线）
- 不同审阅状态用不同背景色标注
- 文件名格式：`{项目名}_审阅结果_{日期}.xlsx`

## 八、配置说明

### 环境变量（.env 文件）

```bash
# 复制模板
cp .env.example .env

# 编辑 .env
DEEPSEEK_API_KEY=sk-xxxx          # DeepSeek API 密钥（腾讯网关用）
DEEPSEEK_MODEL=deepseekv4flash    # 模型名称（腾讯网关的服务 ID）
DEEPSEEK_API_URL=https://tokenhub.tencentmaas.com/v1/chat/completions  # API 地址
SECRET_KEY=change-me-in-production  # Flask 密钥
```

`.env` 已加入 `.gitignore`，不会泄露到 Git。

### 用户管理

用户列表从 `文件说明.xlsx` 的「用户与权限」sheet 读取，字段：英文名、中文名、角色（负责人/校验人）。

### 启动

```bash
python app.py
# → http://localhost:5000
```

修改代码后需手动重启（`use_reloader=False` 关闭了自动重载）。

## 九、已知限制与注意事项

| 限制 | 说明 |
|------|------|
| 🚀 发送功能 | API 效果不如对话框，**推荐用 📋 复制 + 豆包** |
| 无自动重载 | 改代码后需手动重启 Flask |
| 导出格式丢失 | 导出后用 openpyxl 读回会丢失富文本（Excel 本身没问题） |
| 端口占用 | 任务管理器杀 Python 进程 |
| JS 兼容性 | 不能使用 `?.` 可选链语法 |
| v2 链接较少 | Prompt 生成时自动合并官方网站和权威网站的全部链接 |
