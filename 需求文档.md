一、产品概述

一个帮助薪酬合规校验员在线审阅Excel表格、标记状态、协作分工、导出带完整审阅标记的Excel的轻量级Web工具。

用户角色：

| 角色 | 职责 |
|------|------|
| 项目负责人 | 创建项目、上传Excel、分配模块、审阅所有模块、导出结果 |
| 校验人 | 审阅分配给自己的模块、查看团队进度 |

### 解决的痛点

| 痛点 | 解决方案 |
|------|----------|
| 逐项审阅容易遗漏 | 结构化展示，l2模块整体对照，进度追踪 |
| 审阅结果分散，协作困难 | 集中存储，模块级任务分配 |
| 单元格内多类型更改难以标记 | 富文本编辑器，颜色/格式标注增删改 |
| 长文本中嵌入链接难以校验 | 自动提取链接，逐条展示，独立标记状态 |
| 导出结果费时易错 | 一键导出带完整审阅标记的 Excel |
| 分工不明确，重复工作 | 按 l1 模块分配校验人，权限控制 |

### 用户旅程

```
登录 → 首页 → 创建项目 → 上传Excel → 分配模块 → 审阅工作台 → 导出结果
```

二、Excel列分类与审阅对照

2.1 列分类

| 类别 | 列名 | 审阅工作台 | 导出 |
|------|------|-----------|------|
| 审阅对象（6列） | 实务内容（官方/行业通用/内部口径）、参考依据（官方/行业权威/行业常规） | 展示+编辑 | 新增_校验列 |
| 结构信息（4列） | l1_标题、l1_说明、l2_标题、l2_说明 | l2_说明作为审阅范围参考 | 原样保留 |
| 参考信息（3列） | 属性、状态、内部计算链路 | 不展示 | 原样保留 |

2.2 审阅对照关系

| 对照对 | 内容列 | 依据列 | 审阅方式 |
|--------|--------|--------|----------|
| 第1对 | 实务内容（官方） | 参考依据（官方） | 用官方链接核对官方内容 |
| 第2对 | 实务内容（行业通用） | 参考依据（行业权威） | 用行业权威链接核对行业内容 |
| 第3对 | 实务内容（内部口径） | 参考依据（行业常规） | 用行业常规链接核对内部口径 |

判断标准：① 链接能否打开？② 文本信息是否与网页内容一致？③ 哪部分有信息源、哪部分没有？

三、审阅状态与标记规则

3.1 审阅状态（4种）

| 状态 | 颜色 | 单元格背景色 |
|------|------|-------------|
| 待审阅 | 灰色 | #F2F3F5 |
| 已确认 | 绿色 | #36cf5066 |
| 需修改 | 黄色 | #ffb70045 |
| 待讨论 | 紫色 | #722ed1a5 |

3.2 文本标记规则

| 操作 | 标记方式 | 颜色 | 用途 |
|------|---------|------|------|
| 加粗 B | 加粗字体 | — | 强调 |
| 修改 R | 红色文字 | #C00000 | 修改原有内容 |
| 增加 G | 绿色文字 | #548235 | 新增内容 |
| 无源 N | 蓝色文字 | #2E75B5 | 信息无来源 |
| 删除 D | 删除线 | — | 需删除的内容 |

3.3 链接标记

- 链接有效：正常显示
- 链接异常（打不开/内容不符/已过时）：红色文字（#C00000），导出时同步生效
- 有正确替换链接：旧链接红+划掉，新链接绿
- 每个链接提供 [复制] 按钮，点击在右半屏新窗口打开

3.4 查看原文/查看修改

- 内容和参考面板底部均有「查看原文」按钮（有修改时显示）
- 内容面板：查看原文 → 纯文本，查看修改 → 富文本（含颜色格式）
- 参考面板：查看原文 → 原文+链接按钮，查看修改 → 纯文本

四、权限规则

| 角色 | 权限 |
|------|------|
| 项目负责人 | 可编辑所有模块（含已分配）、可分配模块、可上传/删除项目 |
| 已分配校验人 | 可编辑分配给自己的模块 |
| 其他用户 | 只读查看所有模块 |

五、审阅工作台

5.1 布局

```
┌─ 顶栏 ───────────────────────────────────────────────┐
│ « ← 返回  项目 / L1 / L2   第N/总数模块  校验人:XX   │
├─ 左树 ─┬─ 内容区 ───────────────────────────────────┤
│ ▼国家调研│                                              │
│   ├入职申报│ [第1对:官方] [第2对:行业] [第3对:内部]     │
│   └…     │                                              │
│          │ ┌─ 实务内容 ────────────────────────────┐  │
│          │ │ 字段名  [状态标签]  字数  [状态按钮]   │  │
│          │ │ …内容文本…                             │  │
│          │ │ [查看原文]                    [✎编辑]  │  │
│          │ └───────────────────────────────────────┘  │
│          │ ┌─ 参考依据 ────────────────────────────┐  │
│          │ │ 字段名  [状态标签]  链接数  [状态按钮]  │  │
│          │ │ …原文链接可点击打开…  [复制]  [状态]   │  │
│          │ │ [查看原文]                    [✎编辑]  │  │
│          │ └───────────────────────────────────────┘  │
│          │ [◀ 上一模块]              [下一模块 ▶]     │
└──────────┴──────────────────────────────────────────┘
```

5.2 功能要点

- 左侧树折叠/展开，当前模块蓝色高亮，可通过顶栏 « » 按钮折叠/展开整个侧边栏
- 三对通过 Tab 切换，默认显示第1对
- 编辑模式原地替换：点 ✎ → 面板变 Quill 编辑器 → 保存/取消
- 参考依据链接就地渲染为可点击链接，点击在屏幕右半侧新窗口打开
- 内容和参考面板均有「查看原文/查看修改」切换
- 链接可标记状态（有效/打不开/内容不符/已过时）并填写正确链接
- 链接异常标记后导出 Excel 时自动标红
- l2_说明可展开/收起（超过100字截断）
- 非负责人自动跳转到第一个分配给自己模块
- 保存失败自动重试 3 次（0.5s/1.0s/1.5s 递增延迟）
- 所有请求和错误记录到 `app.log`（10MB 自动轮转，保留 3 个备份）

5.3 快捷键

| 按键 | 操作 |
|------|------|
| 1/2/3/4 | 切换审阅状态 |
| →/↓ | 下一个l2模块 |
| ←/↑ | 上一个l2模块 |
| Esc | 关闭编辑器 |

六、协作机制

- 所有用户操作同一个数据库
- 每操作一步立即存盘
- 模块隔离：用户只能编辑自己负责的模块
- 手动刷新获取他人更新

七、用户登录

- 用户列表由 文件说明.xlsx「用户与权限」sheet 定义
- 支持英文名或中文名登录
- 登录页展示所有可用账号

八、导出

- 同一Sheet，每个审阅列右侧插入_校验列
- 校验列内容含颜色格式（绿字/红字/蓝字/删除线/加粗）
- 校验列背景色按状态着色
- 文件名：项目名称_审阅结果_日期.xlsx
- 结构列和参考列原样保留

九、技术栈

| 层面 | 技术 |

十一、部署与运维

- 启动：`python app.py`，浏览器访问 `http://localhost:5000`
- 内网访问：`http://你的IP:5000`（需放行防火墙 5000 端口）
- cpolar 内网穿透可解决公司 WiFi 不稳定问题
- 日志：所有请求记录到 `app.log`，10MB 自动轮转
- 备份：`python backup.py`，定时任务每小时备份一次 `data.db`
- 数据清空：Python 清表 + `VACUUM`

十二、术语表

| 术语 | 说明 |
|------|------|
| 校验列 | 导出 Excel 中新增的列，存储审阅标记后的内容 |
| 审阅字段 | 每个需要审阅的单元格，每 l2 模块 6 个 |
| l1 模块 | 一级分类，如"算薪规则" |

| 校验人 | 被分配审阅特定模块的人 |
| 实务内容 | AI 生成的需要校验的文本 |
| 参考依据 | 对应实务内容的官方/行业信息来源 |
| 无源 | 信息在参考依据中找不到来源，标蓝色 |
