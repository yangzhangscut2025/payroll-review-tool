## 报错复盘总结

### 一、问题现象

| 现象 | 描述 |
|------|------|
| 同事访问 | 之前能打开，后来突然打不开 |
| 自己访问 | 127.0.0.1:5000 和 172.16.103.125:5000 都能正常打开 |
| 防火墙 | 关闭后同事仍然无法访问 |
| ping结果 | 部分同事能通，部分显示"无法访问目标主机" |


### 二、排查过程

#### 阶段1：排除服务本身问题
- 本地访问正常 → 服务在运行，不是代码问题
- 监听地址 `0.0.0.0:5000` → 正确，非 `127.0.0.1`

#### 阶段2：排除IP变化问题
- `ipconfig` 显示 IP 一直是 172.16.103.125 → 不是IP变化导致

#### 阶段3：排除防火墙问题
- 关闭防火墙后仍无法访问 → 不是防火墙规则问题

#### 阶段4：定位真正原因
通过 `netstat -ano | findstr 5000` 发现：

```
TCP    0.0.0.0:5000           0.0.0.0:0              LISTENING       4212
TCP    0.0.0.0:5000           0.0.0.0:0              LISTENING       15780
```

**根本原因**：同一个端口（5000）被两个进程（PID 4212 和 PID 15780）同时占用。

大量的 `TIME_WAIT` 状态连接表明，端口资源被耗尽，导致外部新连接无法被正确处理。


### 三、根本原因分析

| 直接原因 | 根本原因 |
|----------|----------|
| 两个进程占用同一端口 | Flask 被重复启动（或之前调试的进程未完全终止） |
| 大量 TIME_WAIT 堆积 | 端口资源耗尽，新连接无法建立 |


### 四、解决方案

| 步骤 | 命令 | 说明 |
|------|------|------|
| 1 | `taskkill /PID 15780 /F` | 杀掉多余的进程 |
| 2 | `taskkill /PID 4212 /F` | 杀掉多余的进程（如有需要） |
| 3 | `python app.py` | 重新启动 Flask 服务 |
| 4 | `netstat -ano \| findstr 5000` | 确认只有一个 LISTENING |


### 五、关键命令速查

```bash
# 查看端口占用
netstat -ano | findstr 5000

# 查看进程名称
tasklist | findstr PID号

# 强制结束进程
taskkill /PID PID号 /F

# 查看本机IP
ipconfig | findstr IPv4

# 测试网络连通性
ping 172.16.103.125
```


### 六、经验教训

| 类型 | 内容 |
|------|------|
| ✅ 做对了什么 | 本地能访问 ≠ 外部能访问；用 `netstat` 排查端口状态；区分防火墙问题和端口冲突 |
| ❌ 走了什么弯路 | 前期一直在排查防火墙和网络隔离，忽略了端口冲突这个可能性 |
| 💡 关键认知 | 端口被多个进程占用时，本地访问不受影响，但外部连接会被阻塞 |


### 七、预防措施

1. **每次启动前先检查端口**
   ```bash
   netstat -ano | findstr 5000
   ```
   如果已有 `LISTENING`，先杀掉再启动新服务

2. **使用 `Ctrl+C` 正常退出**，不要直接关闭终端

3. **若经常冲突，换端口**：
   ```python
   app.run(host='0.0.0.0', port=5001, debug=True)
   ```

4. **保持单一服务实例**，避免多次启动
